Message

How to dialog with users and how to display different message kinds to them. What and when should they be used?

  • Overview
  • Error message
  • Warning message
  • Information message
  • Confirmation message
  • Multiple messages

Overview

A message is a short meaning content that is communicated and displayed by words and signals. In applications, messages are considered as "feedbacks" responding to end-users, so they play an important role in the development process as well success of applications. To avoid unclear and incorrect messages, you need to clarify usage of message types and how to display them.

Messages are grouped into 4 types:

Error
Warning
Information
Confirmation

In eXo Platform, message(s) can be displayed in a single (only one message) or multiple (more than 1 message) way.

Error message

An error message is to notify users of wrong actions that have been done. Effective error messages should be brief, relevant, clear and courteous about the followings:

Problem - Alert what problems happened. For example: Your connection is time out.
Cause - Explain the reason of problems. For example: Your chosen Username already exists.
Solution - Provide suggestions for users to fix the problem. For example: Please input a valid email format, such as yourname@example.com.

  • Your email address is incorrect. Please input a valid one!

Don'ts in error messages

Incorrect message type - Each type message has its own style to alert. So, it is a need to avoid using inccorect message type. For example, alert "Your username or password input is incorrect" is error message but it is alerted as warning.

Too long content - Most of users want to scan rather than read many things in details.

Blame users - Need to avoid messages that make user feel rude.

Technical error messages - It is recommended to avoid technical messages such as programming content that make end-users confused.

Unknown error messages - Any unknown error messages make users feel your application unreliable, so try to provide a detailed message about invalid actions.

Complicated error messages - Should avoid error messages that report so many problems and causes. An error message should detect one problem or cause.

Warning message

A warning message is to notify users of a situation that might cause a problem in the future. A good warning message should:

Involve risk - Alerts users of something significant. For example: Are you sure to delete the driver and all its data?
Have immediate relevance - Gives clear reasons of warning. For example: Your account will be blocked after 5 times of incorrect password to prevent hackers!
Be clear and specific - Defines sources of warning, potential occurrences, and what users should do. For example: Your password is not strong enough. It should have both texts and numbers.

Template

Warning
  • Your account will be blocked after 5 times of incorrect password to prevent hackers!

Don'ts in warning messages

Redundant messages - Do not show unnecessary and unuseful warning messages that might annoy users.

Overwarning - Too many warning messages will make users doubt about the security of applications.

Information message

An information message is to notify users of useful and relevant, but not urgent information. It is considered as a notification that lets users know specific information. For example: Your account has been registered successfully. Please go to your email address to activate your account.
A good information message should be:
Useful - Let users know something new or have been done successfully.
Optional - Show suggestions (such as version upgrade or new functions) that are not mandatory.
Relevant - Show helpful information that users might not have known.

Infomation

Don'ts in information messages

Redundant - The message includes many redundant information or informs what users already know.
Annoying - The message displays too much that is not necessary. For example, "These selected files have been deleted successfully."
Actionable - The information does not require immediate action.

Confirmation message

A confirmation message is to ask if users want to excute with an action. A confirmation can be displayed in 2 ways: default browser pop-up or customized popup window.
For important changes (for example, deleting a forum that causes all its topics and posts to be lost), the confirmation should be a warning that is displayed in a customized popup rather than browser popup.
A good confirmation message should give the followings:
Obvious result of an action done by the user.
Relevant verification if the user wants to excute with the action.
Easy-to-understand question with two or more responses (for example, OK - Cancel, Yes - No).

Template

Confirmation
  • Are you sure to delete this driver and all its data?
OK Cancel

Confirmation
  • Are you sure to delete this driver and all its data?
OK Cancel

Don'ts in confirmation messages

Unnecessary confirmation - For example, "Are you sure to rename the file?"
Annoying confirmation - For example, there is no change on the form. However, when closing it, there is a confirmation: "Are you sure to close the form? "
Redundant confirmation- For example, some actions we can redo, recover the old content by its revision or from the trash, it is no need to have deletion confirmation.

Multiple messages

eXo Platform allows displaying multiple messages at the same time following the order priority: Error - Warning - Information.

Template

Content of the first displayed message type is expanded by default. Each message type can be clicked to show/hide flexibly. The label of each message type is in the format: {Number} Error(s)/Warning(s)/Information. For example: 5 Errors, 2 Warnings, or 1 Information.

Alert
  • 3 Errors
    • - The field "First Name" is required.
    • - The field "Last Name" is required.
    • - Your email address is invalid. Please enter a different address.

    1 Warning
    • - Your password is not strong enough.

    1 Information
    • - Your registration is completed. Please check your email inbox to activate your account!
OK
