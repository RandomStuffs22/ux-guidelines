Overview

A content can be promoted to decide its quality. The rate function allow to do this.The main purpose of the document is to provide a pattern of rate.

What Problem Does This Solve?

The user needs to know the quality of a content, the rate function not only allow to promote personal evaluation but guide other users about what is good and bad.

When to Use This Pattern?

In a system with large or similar content that need to categorize or evaluate the quality of contents, the rate function is necessary.

What's the Solution?

Template

In case no rate:

 Rate This Topic

My Rating: no value yet

In case already having rate:

 Rate This Topic

My Rating: no value yet

In case rated by current user:

uiIconRatedVote

Markup

HTML is your standard unordered list with links.

 Rate This Topic

My Rating: no value yet

Launch Rate Demo

Status

Status Description
No rate by default The value of "My rating" is "no value yet" and all stars are gray.
Already having rate by the other users The value of "My rating" depend on the number of star. 1 star : Poor, 2 stars : Below average, 3 stars : average, 4 stars : Above average, 5 stars : Good .
Hover/press on stars The star is changed from gray (not rate) and pale yellow (having rate) into yellow. The tooltip is displayed correspondly with number of selected stars.
Rated by the current user The stars are changed into yellow. The value of "My rating" is matched the value of selected stars.

Rules

UP-01 Upload button or symbol is easy to see.
UP-02 Only one file can be selected at once.
RP-03 Multiple files are being uploaded.
RP-04 The uploading percentage number must be matched with the progress bar status.
RP-05 The uploaded files are saved into database only when the Save button are saved.
RP-06 The file title must be cut off with 20 character and "..." in case having long title.

Why to Use This Pattern?

The pattern is to make sure the consistency among applications in the way to uploading files.
