Overview

Pagination refers to numbering pages in a document. On the internet, pagination is used for such things as displaying a limited number of results on search engine results pages, or showing a limited number of posts when viewing a forum thread or a wall. Pagination is used in some form in almost every web application to divide returned data and display it on multiple pages. Pagination also includes the logic of preparing and displaying the links to the various pages.
Full Pagination: Full pagination of pages display as well as the total number of nodes. This is the full mode and the legacy one.
More Mode Pagination: This mode allows to display more content by scrolling mouse to the bottom or clicking on Show More button at the end of a page.

What Problem Does This Solve?

The user needs to view data items from a potentially large set of sorted data that will not be easy to display within a single page.

When to Use This Pattern?

In case displaying all data at once, the page may be loaded very slow. Pagination is a good solution to limit quantity of data display and also make data easy to view and control with page index or show more option. With a page or a form having large data to display, pagination is really necessary.
Use when the content is too long to display at once.
Use when the expected content is prefered to find on first few pages.
Use when the the content is too large to display at once

Stateful page links

Links are customizable and work in a number of circumstances with the right class. .disabled for unclickable links and .active for current page.

Flexible alignment

Add either of two optional classes to change the alignment of pagination links: .pagination-centered and .pagination-right.

Item per page

The default number of item per page of each product should be displayed fully on the desktop screen to avoid t redundant blank space. The default may be fixed with 10 or 20. However, it is nice to have the option to change the number items displaying per page.

Page size:

Depending on the characteristics of the content display, there is defination for suitable maximum number of item per page.
The number of items per page will be loaded depending on the value of the Maximum selected page size.
The number of pagination button will be changed depending on the number of item per page.

More Mode Pagination

In More mode, all data can be displayed in 1 page. However, to avoid overloading, only amount of items are loaded by default. If users want more data to display, they can click on Show More link or scroll the mouse down.

Rules

MPR-01 Non pagination in case all content is loaded by default
MPR-02 Show More link is put at the center bottom of a page
MPR-03 Same number item per Show More click or scroll mouse down
MPR-04 Go Up link is displayed in case all content needs at least 1 Show More click or scroll mouse down

Item per load

With show more mode pagination, all items are loaded in one page. Each click or scroll mouse down shows more items.
The number of items per click on Show More can be fixed in code or there is option in Settings to define number of items loading at once.
10 or 20 items per click or scroll mouse down is ideal depending on your item feature.

Why to Use This Pattern?

We need to use this pattern to improve paginations among applications. The consistency of paginations in use will make your products look more professional and statistical.
