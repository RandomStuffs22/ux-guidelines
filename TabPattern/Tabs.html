Overview

A tab is a resource you can use in the same browser to open up another content. The content of a page that is grouped in different tabs, can be viewed easily.

What Problem Does This Solve?

Switching among tabs helps users to browse more content flexibly without scrolling mouse down or going to the other page.

When to Use This Pattern?

Tabs should be used in cases:

  • The content of a page or a pop up is long
  • There are more 2 sections
The content of each tab can be viewed separate from each other.

What's the Solution?

Provide a horizontal bar or vertical bar of the content area with buttons or links representing tabs. The being viewed tab has its own visual to differentiate with others.
The content of each tab is displayed in the same area with similar structure.

Status

A tab has 2 main statuses:
Status Description
Default status The tabs can be clicked to view its content. There is css effect when mouse hover on it.
Being viewed status The tab is being viewed, its content is displayed. There is no css effect when mouse hover on it.

Rules

TR-01 Different tabs have the same css style
TR-02 Tab content is displayed in the same area
TR-03 Tab title is limited with 20 words
TR-04 By default, there is only one tab being viewed at once
TR-05 No page load when a tab is clicked

Why to Use This Pattern?

Tabs allow to group a long page content logically that makes easy for users to view and interact with. Besides, the same usage makes the consistency among products in the way displaying data in pages or pop-ups.
